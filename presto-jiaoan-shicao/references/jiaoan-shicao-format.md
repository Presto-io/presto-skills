# 实操教案模板格式参考

基于 Presto `jiaoan-shicao` 2.0.0 模板的“教学设计方案（二）”Markdown 格式规范。生成内容时应使用 YAML Front-Matter、标题层级、字段行和列表表达结构，不要手写普通 Markdown 表格来模拟最终 PDF 表格。

版式规则：封面、学习任务分析和学业评价使用纵向 A4；教学活动设计使用横向 A4。每个二级章节之间自动分页。

## YAML Front-Matter

文档必须以 YAML Front-Matter 开头：

```yaml
---
template: "jiaoan-shicao"
course_name: "电工基本技能训练"
course_attribute: "基本技能课程"
textbook_name: "电工基本技能训练指导"
class_name: "机电技术应用 1 班"
total_hours: "8"
teacher_name: "张老师"
use_time: "2026 年 5 月"
---
```

字段说明：

- `template`: 使用 `jiaoan-shicao`。
- `course_name`: 课程名称。
- `course_attribute`: 课程属性，推荐写 `基本技能课程` 或 `工学一体化课程`。
- `textbook_name`: 教材名称。
- `class_name`: 教学班级。
- `total_hours`: 计划总课时。
- `teacher_name`: 教师姓名。
- `use_time`: 使用时间。

注意：不确定的封面字段保留字段名并把值留空，例如 `teacher_name: ""`。可以保守补全用户材料中明显给出的课程主题、学习任务名和课时；不要虚构教材版本、班级、教师、学校名称或日期。模板程序只读取封面字段，`template: "jiaoan-shicao"` 用于外层 Presto 选择模板；只有至少一个封面字段有值时，YAML front matter 才会生成封面页。

`course_attribute` 勾选规则：

| 输入包含 | 封面显示 |
|---|---|
| `基本技能课程` | `☑基本技能课程  □工学一体化课程` |
| `工学一体化课程` | `□基本技能课程  ☑工学一体化课程` |
| 其它或留空 | `□基本技能课程  □工学一体化课程` |

## 文档结构总览

完整输入由四部分组成，顺序固定：

| Markdown 输入                                 | PDF 效果                                                     |
| --------------------------------------------- | ------------------------------------------------------------ |
| YAML cover fields                             | 生成“教学设计方案（二）”封面页                               |
| `template: "jiaoan-shicao"`                   | 让 Presto 选择实操教案模板                                   |
| `course_name`                                 | 封面“课程名称”                                               |
| `course_attribute`                            | 封面“课程属性”，支持 `基本技能课程` 和 `工学一体化课程` 勾选 |
| `textbook_name`                               | 封面“教材名称”                                               |
| `class_name`                                  | 封面“教学班级”                                               |
| `total_hours`                                 | 封面“计划总课时”                                             |
| `teacher_name`                                | 封面“教师姓名”                                               |
| `use_time`                                    | 封面“使用时间”                                               |
| `## 学习任务分析`                             | 生成学习任务分析表                                           |
| `学习任务：...`、`课时：...`、`起止日期：...` | 学习任务分析表顶部字段                                       |
| `### 一、学习任务描述` 到 `### 五、学习资源`  | 学习任务分析表中的固定分析块                                 |
| `## 教学活动设计——主题`                       | 生成教学活动设计章节；PDF 章节标题统一显示“教学活动设计”     |
| `### 学习环节——学习单元`                      | 新建一张活动表，并拆成“学习环节”和“学习单元”                 |
| `#### 活动名称`                               | 表格“教学活动”列，模板自动编号                               |
| `##### 0.5H`                                  | 表格“课时分配”列                                             |
| `#####` 后四个内容块                          | 依次进入“学习内容、学生活动、教师活动、教学方法与手段”       |
| `## 学业评价`                                 | 生成学业评价表                                               |
| `1. 考核项目；考核细则；考核方式`             | 生成一行评价项                                               |
| `小结：...`                                   | 生成评价表末尾“小结”行                                       |
| `{pagebreak}`                                 | 强制换页，并在教学活动表中延续当前结构                       |
| `{pagebreak:weak}`                            | 弱分页                                                       |
| `同上`                                        | 与上一行同列单元格合并                                       |

## 学习任务分析

学习任务分析从 `## 学习任务分析` 开始。标题下先写三个字段行：

```markdown
学习任务：电工基本技能训练
课时：8
起止日期：2026 年 5 月 12 日——2026 年 5 月 15 日
```

字段行说明：

| 字段       | 写法             | 用途                 |
| ---------- | ---------------- | -------------------- |
| 学习任务   | `学习任务：...`  | 学习任务分析表标题项 |
| 课时       | `课时：...`      | 本学习任务总课时     |
| 起止日期   | `起止日期：...`  | 学习任务实施日期     |

字段值不确定时可以留空，但字段行应保留。`起止日期` 会自动去掉形如 `2026 年` 的年份片段，例如 `2026 年 5 月 12 日——2026 年 5 月 15 日` 会显示为 `5 月 12 日——5 月 15 日`。

下面五个 `###` 标题对应固定表格块：

```markdown
### 一、学习任务描述

说明任务情景、任务要求和最终成果。

### 二、学习目标

可以写分条目标，也可以写自然段。

### 三、学习内容

列出安全规范、工具使用、操作步骤和检测要求。

### 四、学生情况分析

说明已有基础、常见困难和教学支持重点。

### 五、学习资源

工量具、设备：...
耗材：...
其它：...
```

五个块都应保留。`### 五、学习资源` 推荐写 `工量具、设备`、`耗材`、`其它` 三行；模板只识别这三个精确键名，缺少某项时让对应内容留空。

## 教学活动设计

教学活动设计从 `## 教学活动设计——主题` 开始。只要二级标题包含“教学活动设计”，PDF 章节标题都会简化为“教学活动设计”；`——` 后面的主题用于帮助 Markdown 可读，不会显示在章节标题中：

```markdown
## 教学活动设计——电工基本技能训练
```

### 学习环节——学习单元

`###` 会新建一张活动表。标题用 `学习环节——学习单元` 表达：

```markdown
### 安全规范与工具准备——识别常用电工工具并完成安全检查
```

对应 PDF：

| 标题片段                         | PDF 位置 |
| -------------------------------- | -------- |
| `安全规范与工具准备`             | 学习环节 |
| `识别常用电工工具并完成安全检查` | 学习单元 |

支持分隔符：`——`、`—`、`-`。推荐中文长横线 `——`。如果不写分隔符，“学习环节”显示完整标题，“学习单元”为空。

#### 活动

每个 `####` 是一个教学活动，进入“教学活动”列，并由模板自动编号：

```markdown
#### 活动一：任务导入与安全规范确认
```

不要为了控制编号手写 `1.`；模板会在“教学活动”列自动添加 `1.`、`2.`。可以保留“活动一”作为活动名称的一部分。

##### 课时

每个 `#####` 是一行课时安排，进入“课时分配”列，标题原样显示。同一个 `####` 下可以写多个 `#####`，用于表达同一活动的多个课时行：

```markdown
##### 0.5H

安全用电要求、个人防护用品穿戴和工位检查要点。

阅读任务书，按检查清单完成工位和工具安全确认。

说明任务标准，演示安全检查流程，提醒通电前确认要求。

任务驱动、示范教学
```

`#####` 后必须按顺序写四个内容块：

| 内容块顺序   | PDF 列         |
| ------------ | -------------- |
| 第一个内容块 | 学习内容       |
| 第二个内容块 | 学生活动       |
| 第三个内容块 | 教师活动       |
| 第四个内容块 | 教学方法与手段 |

前三个内容块可以多行，模板会按行自动编号并在每行末尾加中文分号；“教学方法与手段”建议写短语或短句。

### 分页

`{pagebreak}` 强制换页；如果分页发生在当前活动内部，后续 `#####` 会延续分页前的学习环节、学习单元和教学活动。分页后会重新生成一张活动表，列宽也会按分页后的章节内容重新计算。

```markdown
##### 0.5H

导线剥削长度、端子压接质量和线号标识规范。

完成导线剥削、端子压接和线号标识练习。

示范标准动作，针对压接不牢等问题即时纠正。

示范教学、即时反馈

{pagebreak}

##### 0.5H

继续完成导线剥削、端子压接和线号标识练习。

对照样件复查本组工艺质量。

巡回检查并指出共性问题。

示范教学、即时反馈
```

`{pagebreak:weak}` 生成弱分页，空间不足时更倾向于换页。不确定时优先使用 `{pagebreak}`。

### 同上

只要某个单元格内容包含 `同上`，模板会尝试把它与正上方同列单元格合并。不要在第一行使用 `同上`，因为上方没有可合并内容。`同上` 合并只在同一个活动块的连续课时行内可靠使用，不要把分页后第一行写成 `同上`。

## 学业评价

学业评价从 `## 学业评价` 开始。每一行评价项使用有序列表：

```markdown
1. 安全文明操作；按安全规程完成防护用品穿戴、工位检查和通电申请；过程观察
2. 工具仪表使用；能正确选择并规范使用电工工具和万用表；实操考核
```

每行用中文分号 `；` 分成三段，英文分号 `;` 也能识别，但推荐统一使用中文分号：

| 分段   | PDF 列   |
| ------ | -------- |
| 第一段 | 考核项目 |
| 第二段 | 考核细则 |
| 第三段 | 考核方式 |

模板支持 `1.`、`1、`、`1)`、`1）` 这几种有序列表前缀。默认保留至少五行评价结构，少于五行会补空行，多于五行会继续追加。若一行超过三个分号，第三段之后的内容会合并到“考核方式”列。

末尾写小结：

```markdown
小结：学生能够在安全规范约束下完成照明控制线路连接与检测，达到电工基本技能训练要求。
```

小结应概括学习任务达成情况和后续改进方向；不确定的学习成效不要虚构。

## 常见错误

不要手写普通 Markdown 表格来模拟教案表：

```markdown
| 学习内容 | 学生活动 |
|---|---|
| 内容 | 活动 |
```

这种表格不会被模板当作“教学设计方案（二）”结构。应使用 YAML Front-Matter、`##`、`###`、`####`、`#####`、字段行和有序列表生成对应表格。

不要省略完整章节。即使材料很少，也要保留封面字段、`## 学习任务分析`、`## 教学活动设计` 和 `## 学业评价` 四部分。

不要虚构不确定元数据。未知的 `textbook_name`、`class_name`、`teacher_name`、`use_time` 留空，不虚构教材版本、班级、教师姓名、学校名称或日期。

不要把 `教学设计方案（一）`、`学习任务过程考核评价表` 当作本模板支持的正文部分。这些属于其它表单或后续扩展范围，本参考只教学设计方案（二）。

不要在教学活动表第一行使用 `同上`。第一行上方没有可合并单元格。

## 完整示例

```markdown
---
template: "jiaoan-shicao"
course_name: "电工基本技能训练"
course_attribute: "基本技能课程"
textbook_name: "电工基本技能训练指导"
class_name: "机电技术应用 1 班"
total_hours: "8"
teacher_name: "张老师"
use_time: "2026 年 5 月"
---

## 学习任务分析

学习任务：电工基本技能训练
课时：8
起止日期：2026 年 5 月 12 日——2026 年 5 月 15 日

### 一、学习任务描述

本学习任务围绕常用电工工具识别、安全用电规范、导线剥削与端子压接、照明控制线路连接和通电检查组织训练。学生需要完成工具检查、材料准备、线路连接、绝缘检测和故障排查，形成符合安全规范和工艺要求的实操成果。

### 二、学习目标

1. 能说出常用电工工具、仪表和低压电器元件的名称、用途及安全注意事项。
2. 能依据接线图完成单控照明线路的导线处理、端子连接、绝缘包扎和通电检查。
3. 能使用万用表完成通断检测和简单故障判断，并记录检测结果。

### 三、学习内容

1. 安全用电基本要求、个人防护用品使用和实训工位安全检查。
2. 剥线钳、尖嘴钳、螺丝刀、万用表等工具仪表的识别、选用和维护。
3. 导线剥削、端子压接、接线端固定、绝缘处理和线路整理。
4. 单控照明控制线路的识图、连接、检测、试运行和常见故障处理。

### 四、学生情况分析

学生已学习电工基础中的电路组成、电压电流概念和安全用电常识，但实操经验差异较大。部分学生能读懂简单电路图，但在导线处理、端子压接和检测记录方面规范性不足。

### 五、学习资源

工量具、设备：电工实训台、万用表、剥线钳、尖嘴钳、螺丝刀、验电笔、绝缘胶带、安全防护用品。
耗材：BV 导线、接线端子、灯座、开关、熔断器、线槽、标签纸。
其它：单控照明线路图、安全操作规程、任务书、检测记录表、评价量规。

## 教学活动设计——电工基本技能训练

### 安全规范与工具准备——识别常用电工工具并完成安全检查

#### 活动一：任务导入与安全规范确认

##### 0.5H

电工实训任务要求、安全用电红线、个人防护用品穿戴和工位通电前检查要点。

阅读任务书，按安全检查清单检查工位、工具和防护用品，标记不符合项并完成整改。

说明本次训练成果标准，演示安全检查流程，提醒带电操作禁忌并确认学生分组。

任务驱动、示范教学、清单检查

#### 活动二：工具仪表识别与材料准备

##### 1H

常用电工工具、万用表档位、低压元件和导线规格的识别方法。

分组识别工具和元件，选择适合任务的导线与端子，填写材料准备记录。

结合实物讲解工具用途和仪表使用边界，巡回检查学生选材是否符合任务要求。

实物展示、小组协作、讲练结合

##### 0.5H

导线剥削长度、端子压接质量、绝缘处理和线号标识规范。

完成导线剥削、端子压接和线号标识练习，对照样件自查工艺质量。

示范标准操作动作，针对毛刺、铜芯外露、压接不牢等问题进行即时纠正。

示范教学、即时反馈、过程评价

{pagebreak}

### 线路连接与检测——完成照明控制线路连接和通电检查

#### 活动三：识读线路图并完成接线

##### 1.5H

单控照明线路的电源端、开关端、灯座端连接关系和接线顺序。

根据线路图在实训板上布线、接线和固定导线，完成后进行小组互检。

引导学生分析电流路径，巡回检查接线工艺和安全距离，记录共性问题。

项目教学、小组互检、巡回指导

##### 1H

线槽整理、接线端复查、标识核对和工艺质量自评。

对照工艺标准整理线路，复查端子压接、导线走向和线号标识，填写自评记录。

示范线路整理标准，组织学生按清单复查接线质量并提示常见扣分点。

清单复核、同伴互评、过程评价

#### 活动四：线路检测、通电试验与故障处理

##### 1H

通电前通断检测、绝缘检查、试运行步骤和常见故障判断方法。

使用万用表完成通断检测，确认无误后申请通电试验，记录检测数据和故障处理过程。

审核通电申请，监督试运行过程，指导学生定位断路、虚接和接线错误。

实操考核、问题导向、现场指导

##### 1H

故障模拟、原因分析、排故步骤和复测确认方法。

根据教师设置的典型故障开展排查，说明判断依据，完成复测并修订检测记录。

设置断路、虚接等典型故障，引导学生按“观察—测量—判断—修复”流程处理。

情境模拟、故障诊断、分层指导

##### 1H

检测记录整理、质量评价、改进计划和岗位规范迁移。

完善检测记录，依据评价量规完成小组互评，提出下一次训练的速度和质量改进目标。

组织评价反馈，讲评典型问题，帮助学生把安全规范迁移到后续电工实训任务。

评价反馈、经验迁移、学习反思

##### 0.5H

成果展示、工位整理、工具归位和实训反思。

展示本组线路运行效果，整理工位并提交检测记录和学习反思。

组织成果点评，归纳安全规范和工艺质量要点，布置后续改进任务。

成果展示、反思总结、评价反馈

## 学业评价

1. 安全文明操作；按安全规程完成防护用品穿戴、工位检查和通电申请，操作过程无违规行为；过程观察
2. 工具仪表使用；能正确选择并规范使用电工工具和万用表，测量档位选择正确；实操考核
3. 导线处理质量；导线剥削长度合适、端子压接牢固、绝缘处理规范、标识清楚；作品检查
```
