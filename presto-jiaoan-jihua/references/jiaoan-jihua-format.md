# 授课进度计划表模板格式参考

基于 Presto `jiaoan-jihua` 1.0.2 模板的“工学一体化课程/基本技能课程授课进度计划表”Markdown 格式规范。生成内容时应使用 YAML Front-Matter、`##` 学习任务、`###` 学习环节和 `教学内容-课时` 行表达结构，不要手写普通 Markdown 表格来模拟最终 PDF 表格。

## YAML Front-Matter

文档必须以 YAML Front-Matter 开头：

```yaml
---
major_name: "电气自动化技术"
course_name: "电气设备控制线路安装与调试"
teacher_name: "张老师"
class_name: "29WG电气3"
first_teaching_day: "2026-03-06"
template: "jiaoan-jihua"
---
```

字段说明：

| 字段 | 用途 |
|---|---|
| `major_name` | 表头“专业名称” |
| `course_name` | 表头“课程名称” |
| `teacher_name` | 表头“授课教师”，页脚“制表”优先使用此字段 |
| `class_name` | 表头“授课班级” |
| `first_teaching_day` | 课程排课起点，格式 `YYYY-MM-DD` |
| `template` | 固定写 `jiaoan-jihua` |

注意：不要主动填写 `school_year`、`semester`、`week_range`、`prepared_by`、`daily_hours` 或 `calendar_json`。这些字段由模板根据内置校历、教师姓名和排课结果推断；旧文档里存在这些字段时仍兼容。可以保守补全用户材料中明显给出的课程主题、学习任务名和课时；不要虚构专业、班级、教师或日期。

## 文档结构总览

完整输入由两部分组成，顺序固定：

| Markdown 输入 | PDF 效果 |
|---|---|
| YAML fields | 生成标题、专业名称、课程名称、授课教师、授课班级 |
| `template: "jiaoan-jihua"` | 让 Presto 选择授课进度计划模板 |
| `## 学习任务名称` | 生成“学习任务N名称”行和任务总学时 |
| `### 学习环节名称` | 生成合并单元格“学习环节N名称” |
| `教学内容-4` | 生成教学内容行，学时列显示 `4` |
| `教学内容` | 生成教学内容行，学时列默认为 `2` |

模板输出的表格列为：学习环节、教学内容、周次、星期、学时。周次和星期由模板自动计算，不需要手写。任务汇总行的学时显示为 `24H` 这类形式，教学内容行的学时列只显示数字。

## 学习任务

每个学习任务从 `##` 开始：

```markdown
## CA6140卧式车床电气控制线路安装与调试
```

对应 PDF：

- 左侧显示 `学习任务1名称：`
- 右侧显示学习任务名称
- 学时列显示该任务下所有教学内容课时合计，如 `24H`

多个 `##` 会按出现顺序生成“学习任务1名称”“学习任务2名称”等，不需要手动编号。

## 学习环节

每个学习环节从 `###` 开始，必须放在某个 `##` 学习任务下：

```markdown
### CA6140卧式车床电气控制线路识读
```

对应 PDF 中的学习环节列会显示 `学习环节1名称：CA6140卧式车床电气控制线路识读`，并对该环节下的多行教学内容做纵向合并。

如果用户只给了教学内容，没有学习任务或学习环节，应补出保守占位结构：

```markdown
## 学习任务名称

### 学习环节名称

教学内容-2
```

## 教学内容与课时

教学内容写在 `###` 下，每行一条：

```markdown
低压电器元件检测与安装-4
控制线路布线与通电调试-6
安全操作记录与质量检查-2
```

课时解析规则：

| 输入 | 解析结果 |
|---|---|
| `安技教育-1` | 教学内容 `安技教育`，课时 `1` |
| `控制线路布线与通电调试-6` | 教学内容 `控制线路布线与通电调试`，课时 `6` |
| `安全教育` | 教学内容 `安全教育`，课时默认 `2` |
| `PLC 控制模板讲解-2` | 教学内容可包含空格，课时 `2` |

稳定写法是 `教学内容-正整数`。不要写小数课时、中文课时单位或 `H` 后缀；这些形式不会被模板可靠识别。教学内容本身需要包含连字符时，模板按最后一个 `-数字` 解析；例如 `PLC-1 控制模板讲解-2` 会解析为教学内容 `PLC-1 控制模板讲解`、课时 `2`。

## 日历与排程

模板根据以下信息自动排程：

- `first_teaching_day`: 课程排程起点。
- 内置学校校历：校历第一天用于推断学年和学期，并作为周次计算锚点。
- 默认每日课时：8 课时。

兼容旧输入中的外部日历，推荐使用只包含上课日期的数组：

```json
[
  "2026-03-06",
  "2026-03-07",
  "2026-03-09"
]
```

排程规则：

- 内置或外部有效日历中只使用 `workday: true` 的日期。
- `school_year` 从校历第一天推断：上半年为上一年到当年，下半年为当年到下一年。
- `semester` 从校历第一天推断：上半年为第二学期，下半年为第一学期。
- `week_range` 从实际排课覆盖的周次范围推断。
- 外部日历 JSON 解析失败、JSON 为空数组或任一日期无法按 `YYYY-MM-DD` 解析时，PDF 中会显示“日历文件解析失败，已使用默认日历”。
- 如果有效日历天数不足，模板会从日历最后一天继续向后生成周一至周五工作日补足。
- 一条教学内容跨多天时，周次和星期列用空格连接，例如 `1 2`、`5 1 2`。

## 常见错误

不要手写普通 Markdown 表格：

```markdown
| 学习环节 | 教学内容 | 学时 |
|---|---|---|
| 环节 | 内容 | 2 |
```

这种表格不会被模板当作授课进度计划结构。应使用 `##`、`###` 和 `教学内容-课时` 行。

旧式表格或其他正文中没有任何 `##` 学习任务时，模板会生成占位任务并显示提示：“请使用 ## 学习任务 和 ### 学习环节 输入授课计划”。因此转换用户材料时应主动改写为三级结构，而不是把原表格原样塞进正文。

不要把周次、星期写进正文行。模板会自动计算周次和星期；正文只写教学内容与课时。

不要省略 `template: "jiaoan-jihua"`。缺少该字段时，外层工具可能无法选择正确模板。

不要虚构不确定元数据。未知的专业、班级、教师、学年学期、开课日期留空；只有用户给出或可从上下文明确推断时才填写。

不要在一行写多个教学内容。每条教学内容单独一行，便于模板逐行排程和合计课时。
