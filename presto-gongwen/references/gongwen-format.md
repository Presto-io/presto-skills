# 公文模板格式规范

基于 GongWen-Template-Lite（类公文模板）的 Markdown 格式规范。

## YAML Front-Matter

文档必须以 YAML front-matter 开头：

```yaml
---
title: "文档标题"
author:
  - "作者1"
  - "作者2"
date: "2026-01-23"
signature: true
template: "gongwen"
---
```

字段说明：

- `title`: 文档标题。短标题（20 字以内）直接写纯文本；只有标题确实过长需要换行时才用 `|` 在语义断点分隔，如 `"项目名称|建设项目立项书"`。
- `author`: 作者列表。
- `date`: 日期（YYYY-MM-DD 格式）。
- `signature`: 是否显示签名区域（true/false）。`true` 时文末右侧自动落款，标题下不显示作者；`false` 时作者显示在标题下方，文末不自动落款。
- `template`: 使用 `gongwen`。

注意：不要使用反斜杠给标题换行；不要在正文里用 `##` 重复 YAML 标题。

## 标题层级与编号

### 一级标题

**不使用 Markdown 一级标题（`#`）**。文档标题由 YAML `title` 字段定义。

### 二级标题（##）

用于主要章节，自动编号为“一、”“二、”“三、”。

```markdown
## 项目概述
## 功能特点
```

### 三级标题（###）

用于子章节，自动编号为“（一）”“（二）”“（三）”。

```markdown
### 项目背景
### 技术路线
```

### 四级标题（####）

用于细分内容，自动编号为“1.”“2.”“3.”。

```markdown
#### 标题层级处理
#### 自动编号机制
```

### 五级标题（#####）

用于最细粒度，自动编号为“（1）”“（2）”“（3）”。

```markdown
##### 无缩进段落
##### 无缩进行内文本
```

转换时不要手动写编号。原文若带 `1.1`、`2.2` 等编号，转为模板标题前先移除数字前缀。

## 段落格式

### 首行缩进

文档自动全局缩进 2 个字符宽度。首行若以冒号开头则不缩进。

### 无缩进段落

使用 `::: {.noindent}` 包裹：

```markdown
::: {.noindent}

这是一段无缩进的段落。

:::
```

### 无缩进行内文本

使用 `{.noindent}` 标签：

```markdown
这是一段无缩进的行内文本。{.noindent}
```

### 无缩进标题

```markdown
#### 这是一个无缩进的标题 {.noindent}
```

### 字符图与复杂结构

复杂 ASCII 框图、字符图或艺术化流程图不应原样保留，应转换为列表、步骤说明或表格。例如：

```markdown
| 环节 | 说明 |
|:-----|:-----|
| 输入 | 收集原始材料 |
| 处理 | 提炼结构并转换格式 |
| 输出 | 生成 Markdown 与 PDF |

: 文档处理流程
```

## 列表格式

### 有序列表

标准缩进：

```markdown
1. 有序列表项，长文本会自动对齐左侧版芯边缘；
2. 第二项。
```

无缩进列表：

```markdown
::: {.noindent}

1. 无缩进有序列表项；
2. 第二项。

:::
```

### 无序列表

```markdown
- 无序列表项，长文本同样自动对齐；
- 第二项。
```

## 换行与分页

### 换行

使用 `{v}` 标记空 1 行。`{v}` 前后必须各有一个空行，使其成为独立段落，否则可能被当作文本输出：

```markdown
这段文字正常。

{v}

这是另一段内容。
```

使用 `{v:N}` 标记空 N 行：

```markdown
这段文字正常。

{v:3}

这是隔 3 行后的内容。
```

### 分页

使用 `{pagebreak}` 标记分页：

```markdown
{pagebreak}
```

使用 `{pagebreak:weak}` 标记弱分页：

```markdown
{pagebreak:weak}
```

弱分页倾向于分页，但不是绝对强制，适合“空间不够就换页”的场景。不确定时优先使用 `{pagebreak}`。

## 图片格式

### 单张图片

自动居中并限制在版心内。单图图题主要取图片文件名（不含扩展名），所以图片文件名要起清楚。图片前后都要有文字，形成“文字-图片-文字”的三明治结构，避免图片独占半页后文字不足留白：

```markdown
这里用两到三行文字说明图片背景、场景或需要读者关注的问题。

![图片说明](图片文件名.jpg)

这里用两到三行文字解释图片要点，并自然过渡到后续内容。
```

没有图注时可以省略 alt text：

```markdown
![](图片文件名.jpg)
```

### 多图并排

连续放置图片即可自动并排：

```markdown
![](图片1.jpg) ![](图片2.jpg)
```

每张图仍按图片文件名生成图题，适合对比图、流程图组合。

### 子图组合

多图写在同一段，且使用相同 alt text，可组成子图组合：

```markdown
![安全隐患整改前后对比](整改前.png) ![安全隐患整改前后对比](整改后.png)
```

PDF 中总图题为 alt text，子图标为 `(a)`、`(b)`，子图小标题取文件名。

### OpenClaw 图片路径

OpenClaw 管道编译时应加 `--root /root/.openclaw/workspace`，图片路径使用相对于工作区根目录的路径：

```markdown
![现场照片](documents/20260510 项目名/photo.png)
```

## 表格格式

标准 Markdown 表格必须有表头行。表格后加 `: 标题` 自动生成“表 n 标题”编号。

```markdown
| 阶段 | 时间 | 负责部门 |
|:-----|-----:|:---------|
| 自查自纠 | 3月1日-15日 | 各部门 |
| 集中检查 | 3月16日-31日 | 安全管理处 |

: 检查工作进度安排
```

表题必须紧跟在表格后面，写法是英文冒号 `:` 加一个空格。不要自己写“表 1”，模板会自动编号。

对齐规则：

| 内容类型 | 对齐方式 | 语法 |
|---------|---------|------|
| 纯数字、日期、金额 | 右对齐 | `---:` |
| 短文本（少于 5 字） | 居中 | `:---:` |
| 长文本、描述性内容 | 左对齐 | `:---` |

表格内换行用 `<br>` 标签：

```markdown
| 项目 | 说明 |
|:-----|:-----|
| A | 第一行<br>第二行 |
```

表格前后也要有文字说明，避免表格孤立造成大段留白。

长表格过长时会自然跨页，后续页重复表头，并显示续表标题。

## 排版避坑指南

### 图片位置：三明治原则

图片不能单独占半页，必须前后都有文字填充：

```text
不推荐：

## 某章节
[大图 - 占半页]
一点点文字
....... 大段空白 .......

推荐：

## 某章节
[段落文字 - 3 到 5 行]
[大图 - 占半页]
[段落文字 - 3 到 5 行]
[自然过渡到下一段]
```

操作要点：

- 图前写 3 到 5 行引入文字，描述背景或提出问题。
- 图后紧跟 3 到 5 行延伸文字，分析细节或补充说明。
- 如果章节内容本身不够长，考虑合并相邻章节。

### 表格位置：同上

表格也遵循三明治原则，且用 `: 标题` 自动编号后尽量让表格和后续文字在同一页。

## 常用 Markdown

公文模板支持常见 Markdown：

| 输入 | PDF 效果 |
|------|----------|
| `**重点**` | 加粗 |
| `*强调*` | 强调样式 |
| `` `signature: true` `` | 代码样式 |
| `[Presto 官网](https://presto.c-1o.top)` | 可点击链接文字 |
| `> 本通知自发布之日起执行。` | 引用 |
| 正文中的 `---` | 横线 |
| `<!-- 注释 -->` | 不显示在 PDF 中 |

## 中文标点自动转换

模板会自动转换常见中文标点，例如英文分号、括号、普通冒号等会转换为中文语境下更合适的符号。URL、特殊标签和 `12:30` 这样的时间不会被改坏。

生成 Markdown 时仍建议直接使用规范中文标点，不把自动转换当作主要写作方式。

### 标题换行：按需使用

- 标题 20 字以内 → 不换行。
- 标题超过 20 字 → 用 `|` 在语义断点处换行。

### 文体收尾

报告、申请类可用：

```markdown
以上报告，请审阅。
```

通知、纪要、发言稿、学习心得、学习笔记类不要强行添加报告式收尾，应按文体自然结束。

## 称呼与落款

文档开头使用称呼：

```markdown
各位领导：
```

或其他适当称呼：

```markdown
尊敬的领导：
各有关部门：
各位同事：
```

## 完整示例

```markdown
---
title: "工作报告"
author:
  - "张三"
date: "2026-03-31"
signature: true
template: "gongwen"
---

各位领导：

为了全面梳理第一季度工作进展，现将有关情况报告如下：

## 工作概述

### 主要成果

本季度各项重点任务总体推进顺利，关键指标保持稳定增长，协同机制进一步完善。

| 指标 | 数值 | 变化 |
|:-----|-----:|:-----|
| 完成率 | 98% | +5% |
| 效率 | 95% | +3% |

: 第一季度关键指标

从数据看，项目完成率和执行效率均有所提升，为后续阶段打下了较好基础。

### 存在问题

（略）

## 下一步计划

::: {.noindent}

1. 重点推进...；
2. 持续优化...。

:::

以上报告，请审阅。
```

## 转换要点

将任意文档转换为公文格式时，需确保：

1. 添加 YAML front-matter，从原文档提取标题、作者、日期信息。
2. 调整标题层级，将原标题映射为二至五级标题，确保编号规则正确。
3. 移除一级标题和手工编号，避免与模板自动编号重复。
4. 规范化段落，正文首行缩进由模板自动处理。
5. 处理图片和表格，补齐前后说明文字并添加必要图注/表注。
6. 根据文体添加合适称呼、落款和收尾，不机械套用报告格式。
