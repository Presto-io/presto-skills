# 公文模板格式规范

基于 GongWen-Template-Lite（类公文模板）的 Markdown 格式规范。

## YAML Front-Matter

文档必须以 YAML front-matter 开头：

```yaml
---
title: "文档标题|可换行"
author:
  - "作者1"
  - "作者2"
date: "2026-01-23"
signature: true
---
```

字段说明：
- `title`: 文档标题，使用 `|` 可强制换行
- `author`: 作者列表
- `date": 日期（YYYY-MM-DD 格式）
- `signature`: 是否显示签名区域（true/false）

## 标题层级与编号

### 一级标题

**不使用 Markdown 一级标题（`#`）**。文档标题由 YAML `title` 字段定义。

### 二级标题（##）

用于主要章节，自动编号为 "一、" "二、" "三、"...

```markdown
## 项目概述
## 功能特点
```

### 三级标题（###）

用于子章节，自动编号为 "（一）" "（二）" "（三）"...

```markdown
### 项目背景
### 技术路线
```

### 四级标题（####）

用于细分内容，自动编号为 "1." "2." "3."...

```markdown
#### 标题层级处理
#### 自动编号机制
```

### 五级标题（#####）

用于最细粒度，自动编号为 "（1）" "（2）" "（3）"...

```markdown
##### 无缩进段落
##### 无缩进行内文本
```

## 段落格式

### 首行缩进

文档自动全局缩进 2 个字符宽度。首行若以冒号开头则不缩进。

### 无缩进段落

使用 `::: {.noindent}` 包裹：

```markdown
::: {.noindent}

这是一段无缩进的段落。

:::
```

### 无缩进行内文本

使用 `{.noindent}` 标签：

```markdown
这是一段无缩进的行内文本。{.noindent}
```

### 无缩进标题

```markdown
#### 这是一个无缩进的标题 {.noindent}
```

## 列表格式

### 有序列表

标准缩进：

```markdown
1. 有序列表项，长文本会自动对齐左侧版芯边缘；
2. 第二项。
```

无缩进列表：

```markdown
::: {.noindent}

1. 无缩进有序列表项；
2. 第二项。

:::
```

### 无序列表

```markdown
- 无序列表项，长文本同样自动对齐；
- 第二项。
```

## 换行与分页

### 换行

使用 `{v}` 标记空 1 行：

```markdown
{v}

这是另一段内容。
```

使用 `{v:N}` 标记空 N 行：

```markdown
{v:3}

这是隔 3 行后的内容。
```

### 分页

使用 `{pagebreak}` 标记分页：

```markdown
{pagebreak}
```

## 图片格式

### 单张图片

自动居中，最大宽度/高度 13.4cm，自动提取文件名作为图注：

```markdown
![](图片文件名.jpg)
```

### 多图并排

连续放置图片即可自动并排：

```markdown
![](图片1.jpg) ![](图片2.jpg)
```

### 子图组合

第一张图片添加说明文字，自动生成 (a)、(b) 编号：

```markdown
![子图说明](图片1.jpg) ![](图片2.jpg) ![](图片3.jpg)

这里可以添加整组图片的说明文字。
```

## 称呼与落款

文档开头使用称呼：

```markdown
各位领导：
```

或其他适当称呼如：

```markdown
尊敬的领导：
各有关部门：
```

## 完整示例结构

```markdown
---
title: "工作报告|2026年第一季度"
author:
  - "张三"
date: "2026-03-31"
signature: true
---

各位领导：

为了...现将有关情况报告如下：

## 工作概述

### 主要成果

1. 完成了...；
2. 实现了...。

### 存在问题

（略）

## 下一步计划

::: {.noindent}

1. 重点推进...；
2. 持续优化...。

:::

{pagebreak}

## 附录

![附录图表示例](附录图片.png)

以上报告，请审阅。
```

## 转换要点

将任意文档转换为公文格式时，需确保：

1. **添加 YAML front-matter**：从原文档提取标题、作者、日期信息
2. **调整标题层级**：将原标题映射为二至五级标题，确保编号规则正确
3. **规范化段落**：移除一级标题，正文首行缩进自动处理
4. **处理图片**：保留图片引用，必要时添加图注
5. **添加称呼与落款**：根据文档类型添加适当的称呼语和结束语
6. **特殊格式**：识别原文中的列表、表格等，转换为对应格式
