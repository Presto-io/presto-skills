# Presto Agent Skills

Official monorepo for [Presto](https://github.com/Presto-io/Presto) Codex agent skills.

## Repository Structure

Each skill lives at the repo root in its own directory:

```
presto-gongwen/           ← skill directory (name = skill name)
  SKILL.md                ← skill definition & instructions
  agents/openai.yaml      ← UI metadata for Codex
  assets/                 ← icons and brand assets
  references/             ← supplementary docs the skill references
```

Additional skills are added as sibling directories at the repo root — no nesting under a `skills/` subfolder.

> **Note:** `.skill` archive files are **not** committed to this repo. They are derivative artifacts that can be generated by CI if needed.

## Installing a Skill

Use the Codex skill installer:

```bash
npx --yes skills add Presto-io/presto-skills --skill presto-gongwen -g -y
```

Replace `presto-gongwen` with the directory name of the skill you want to install.

## Updating a Skill

Use the Skills CLI update command:

```bash
npx --yes skills update presto-gongwen -g -y
```

Restart the agent client after updating so it reloads the new `SKILL.md`.

## CI Automation

- **Validation** — Every PR and push runs `python3 scripts/validate.py` to check skill structure, frontmatter, and metadata.
- **Registry** — Pushes to `main` automatically generate `registry.json` with each skill's frontmatter version and content hash, then deploy it to the CDN endpoint via the `registry-deploy` repository.

## Local Validation

Before opening a PR, run:

```bash
python3 scripts/validate.py
```

This checks all root-level skill directories for:
- Valid `SKILL.md` with required frontmatter (`name`, `description`)
- Non-empty Markdown body
- Valid `agents/openai.yaml` with required `interface` fields
- Resolvable icon asset paths
- Intact reference links

## Adding a New Skill

1. Create a new directory at the repo root with your skill name (lowercase, hyphenated)
2. Add `SKILL.md` with proper frontmatter
3. Add `agents/openai.yaml` with UI metadata
4. Add icon assets to `assets/`
5. Run `python3 scripts/validate.py` locally
6. Open a PR
